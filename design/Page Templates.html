Page templates — the recurring layout skeletons. Pick a template, fill the slots, and the page is consistent without reinventing structure. Pair with the component reference.
Structural Rules · Every Page
Global nav (gnav) is always present, always sticky top:0, always dark ink. Same links on every page.
Max-width: 1140px, centered. Page padding: 28px (--sp-2xl). All content inside this container.
Blueprint grid (28px, 3.5% ink) is always visible as a body::before pseudo-element.
Lifecycle rail appears on every Layer 3 (refinement) screen. The current step is active (terra solid); completed steps are done (green); future steps are faint (dashed). The rail IS the back/forward navigation — every chip is a link.
Carried-bet reminder appears on every screen after the bet front door — a compact locked-mini card showing what was committed. Read-only; links back to edit.
Narrator band appears ONLY on walkthrough/demo screens (Full Flow, Decomposition Visualizer). Never on the product screens themselves. Dark ink, not paper.
Footer with cross-links to related screens. Dashed rule, 11px ink-soft.

Template Decision Tree
Is the user editing a single bet with discipline alongside? → A · Form + Sidebar
Is the user comparing locked commitment to outcome? → B · Read + Act
Is the user scanning/managing many items (ledger, timeline, chains)? → C · Full-Width Data
Is the user providing raw input to be structured? → D · Input + Output
Is this a multi-screen walkthrough for demo/pitch? → E · Storyboard
Common Patterns Across Templates
Carried state. Every screen after the Bet Front Door should show what's been committed upstream. Use the locked-mini component. Read-only; clicking it navigates back to that step.
Controls bar. When a screen has filters, view switchers, or mode toggles, place them in a single row above the main content. Use the seg component for mutually-exclusive modes; chip buttons for filters. Never put controls in the sidebar.
Callouts. For warnings (contention, missing fields, integrity issues), use the callout component below the relevant content area. Icon (⚠/✓/✕) + text. Amber for caution, green for ok, terra for failure.
Transitions. When the user moves between lifecycle steps, the rail updates and the carried-bet reminder reflects the new committed state. No page transition animation; instant swap. Persist scroll position via sessionStorage if returning.
Responsive. Two-column layouts collapse to single-column at 820px. The sidebar drops below the main content. The gnav stays. Timeline views get overflow-x: auto.
