# Dispatch conventions for alphabeta

This file is auto-included as `--read` context on every `dispatch` invocation in this project. Distilled from real failures during Sprint 1 + Sprint 2. Read before editing any file in `apps/web/`.

## TypeScript

- **Strict mode is non-negotiable.** Every file must pass `npx tsc --noEmit` with no errors.
- **Never `any`.** Use `unknown` and narrow with type guards. Example pattern:
  ```ts
  function isFeasibilityInstrument(s: unknown): s is FeasibilityInstrument {
    return s === "ab" || s === "quasi" || s === "observational" || s === "holdback";
  }
  // Then: if (isFeasibilityInstrument(val)) { setDeclared(val); }
  ```
- **`Record<string, unknown>` fields require narrowing.** `bet.instrument.feasibility` is `Record<string, unknown>`. Don't do `setRandomize(f.randomize)` ŌĆö that fails strict tsc because `f.randomize` is `unknown`. Narrow first with a type guard.
- **Subset vs superset types.** `Bet.instrument.type` is `InstrumentType = "ab" | "quasi" | "observational" | "holdback" | "interviews"`. `FeasibilityInstrument` is the subset without `"interviews"`. Use a guard before assigning across the boundary; the persisted type can be `"interviews"` even if the current screen only handles the four quantitative ones.

## Next.js 16 + static export

- **`useSearchParams` MUST be inside a child component wrapped in `<Suspense>`.** The page's default export wraps the inner component:
  ```tsx
  export default function MyPage() {
    return (
      <Suspense fallback={<div className="ab-wrap" />}>
        <MyPageInner />
      </Suspense>
    );
  }
  function MyPageInner() {
    const searchParams = useSearchParams();
    const id = searchParams.get("id");
    // ŌĆ”
  }
  ```
  Do NOT call `useSearchParams()` in the parent and pass its result as a prop.
- **Always add `'use client'` at the top** of any file that uses React hooks (`useState`, `useEffect`, `useSearchParams`, etc.).
- **Don't use `revalidate`, `dynamic`, or `headers()`** ŌĆö they break `output: 'export'`.

## URL shape

- **All bet URLs include `?id=<uuid>`.** Never link to bare `/bet/wager` or `/bet/lock` ŌĆö always `/bet/wager?id=${id}` etc.
- **All inter-stage links are under `/bet/`.** Don't construct `/wager` or `/lock` ŌĆö use the full `/bet/<stage>` prefix.
- Spine-rail stage names are single-word: `wager`, `instrument`, `criteria`, `lock`, `revisit`.

## Tailwind / CSS

- **No `rounded-*` classes.** The design uses no rounded corners. If you see yourself reaching for `rounded-full`, `rounded-md`, etc., stop ŌĆö use straight edges.
- **No `shadow-*` classes.** No drop shadows in this design system.
- **Border widths are `1.5px`** via arbitrary-value Tailwind: `border-[1.5px]`. Default `border` is 1px which doesn't match the design.
- **Dashed borders are the default.** Use `border-dashed`. Solid borders only on active/selected states or the locked-bet card.
- **Use the existing token classes:** `text-ink`, `text-ink-soft`, `text-ink-faint`, `text-terra`, `text-green`, `text-amber`, `text-plinth`, `bg-paper`, `bg-paper-veil`, `bg-terra-soft`, `border-rule`, `border-rule-faint`, etc. They're defined as `@theme` colors in `apps/web/app/globals.css`.
- **Component anatomy lives in `apps/web/app/components.css`.** Classes like `.btn`, `.dashed-panel`, `.scard`, `.heard`, `.wager`, `.tok`, `.spine-rail`, `.srail`, `.annot`, `.st-*` are plain CSS using `@theme` tokens. Reuse them; don't recreate the same anatomy with utility classes.

## React conventions

- **Components defined inside other components get unmounted/remounted on every render** ŌĆö define helpers at module scope. (Hard-learned: the Bet Front Door's `Tok` initially caused input-focus loss because it was a nested function component.)
- **`useEffect` hydration pattern:** use a `cancelled` flag for cleanup:
  ```tsx
  useEffect(() => {
    let cancelled = false;
    (async () => {
      const row = await getBet(id);
      if (cancelled) return;
      // ŌĆ” setState(ŌĆ”)
    })();
    return () => { cancelled = true; };
  }, [id]);
  ```

## Persistence

- **Never call `getDb()` directly from pages.** Use the query layer: `mintDraft`, `getBet`, `listBets`, `updateDraft`, `lockBet`, `recordResolution` from `@/lib/bet/queries`.
- **Immutability is enforced at the query layer:** `updateDraft` rejects post-lock; `lockBet` rejects already-locked. Catch rejections silently ŌĆö they're expected when racing against another tab.
- **Mirror state changes to Dexie via `updateDraft` in a `useEffect`** that depends on the state fields. Guard with a `hydrated` flag so the initial empty state doesn't overwrite the loaded record.

## Imports

- **Path alias `@/`** maps to `apps/web/`. Use `@/lib/...`, `@/components/...`, `@/app/...` ŌĆö never relative paths like `../../../lib/...`.
- **Type-only imports use `import type`:** `import type { Bet } from "@/lib/db/types";` ŌĆö keeps the runtime bundle smaller.

## Testing

- **TDD for new `lib/*` modules.** Tests first, implementation second.
- **Component tests use RTL.** `@testing-library/react`, `screen.getByText`, etc. `vitest.setup.ts` wires `expect` extensions.
- **Use `fireEvent.change` for range-input value changes** ŌĆö jsdom's keyboard handling on range inputs is unreliable.
- See `apps/web/components/bet/__tests__/WagerStatic.test.tsx` and `apps/web/lib/bet/__tests__/queries.test.ts` for patterns.

## Anti-patterns spotted in past dispatches

- Inline JSX-helper components defined inside the page (unmount-remount cycle).
- `setState(undefined)` or `setState({})` on a non-nullable-typed slot.
- Forgetting the `'use client'` directive on a file using hooks.
- Calling `updateDraft(id, ...)` without checking `id !== null` first.
- Putting the right-column annotation **outside** the `<div className="ab-cols">` wrapper.
- Adding `rounded-full` to status badges or pills (no rounded corners, ever).
- Using `bet.instrument.type` as if it were `FeasibilityInstrument` without a guard.
